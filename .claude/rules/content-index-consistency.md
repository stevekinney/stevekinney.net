# Content Index Consistency

`content-index.json` drives the post listing, routing, and sorting on the website. When it disagrees with the canonical sources (`manifest.json` and markdown frontmatter), posts break silently—wrong URLs, wrong dates, or outright crashes.

## Slug must match the markdown filename

The `slug` field in `content-index.json` is used to resolve the markdown module at runtime via `loadWritingMarkdown(slug)`. If it doesn't match the filename (minus `.md`), the page crashes with `MarkdownModuleNotFoundError`.

- The canonical slug lives in `manifest.json` (derived from the filename).
- Always cross-check: `content-index.json` slug === `manifest.json` slug === markdown filename stem.

## Dates must match frontmatter

The `date` and `modified` fields in `content-index.json` control display order and the date shown on listing pages. Use the values from the markdown frontmatter (or `manifest.json`, which mirrors them).

## When regenerating content-index.json

If the file is generated by a script, run the script rather than editing by hand. If editing by hand, diff against `manifest.json` to confirm every field matches.
