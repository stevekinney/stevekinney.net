## Hi, I'm Steve.

I'm a **software engineer**, **educator**, and **engineering leader** based in Denver. Most recently, I spent a year as an Entrepreneur in Residence at [Redpoint](https://www.redpoint.com), where I focused on building agentic workflows and tools. That work included a durable agent execution workflow platform, an IndexedDB-based vector database that runs in the browser, a robust agent tool management platform, and an agent runtime embedded in a Chrome extension.

Before that, I led the Developer Tools team at [Temporal](https://temporal.io). I was one of the early employees, built the [open source Temporal UI](https://github.com/temporalio/ui) and [Temporal Cloud](https://temporal.io/cloud) UI from the ground up, and then built the team responsible for them. For a stretch, that team also owned the CLI and the Visual Studio Code extension, which meant thinking about the entire developer workflow, not just the interface layer.

Earlier in my career, I was the _very first_ Front-End Architect™ at [Twilio](https://twilio.com) and [SendGrid](https://sendgrid.com). Those roles taught me how to build consensus across large organizations, drive architecture through influence, and help teams make good technical decisions at scale.

I still care deeply about front-end architecture, but my work has never really been _just_ frontend. The common thread through all of it is making complicated systems usable: distributed systems, developer platforms, and now AI systems. I tend to live at the intersection of product thinking, technical architecture, and developer experience.

That also shows up in my writing. Lately, I've been researching and publishing on [agent loops](/writing/agent-loops), [memory systems for AI agents](/writing/agent-memory-systems), [AI gateways and durable workflow systems](/writing/ai-gateway-durable-workflows), [prompt engineering across the OpenAI, Anthropic, and Gemini APIs](/writing/prompt-engineering-frontier-llms), [MCP Apps and AI tooling](/writing/mcp-apps), and [what it takes to run untrusted code safely](/writing/designing-a-system-to-run-untrusted-code).

Today, I'm an **instructor** at [Frontend Masters](https://frontendmasters.com/?utm_source=kinney&utm_medium=social&code=kinney), where I teach courses including [Open Source AI with Python & Hugging Face](https://frontendmasters.com/courses/python-ai/?utm_source=kinney&utm_medium=social&code=kinney) and [Cursor & Claude Code: Professional AI Setup](https://frontendmasters.com/courses/pro-ai/?utm_source=kinney&utm_medium=social&code=kinney). I also founded the Front-End Engineering program at [Turing School](https://turing.edu).

I actually started my career in public education. I was an NYC public school teacher in high-need schools across Manhattan, Brooklyn, and Queens. I technically had my principal's license, but I never ended up using it.

I've co-founded [DinosaurJS](https://dinosaurjs.org), a JavaScript conference in Denver, spoken at events like JSConf and RailsConf, and authored "[Electron in Action](https://bit.ly/electronjs)"—though technology moves faster than books, so consider that a historical artifact at this point.
